The test results below show a comparison of recent blocks between the full block processing of Core, the full processing of Bitcrust, as well as the order processing of Bitcrust.

The comparison is between a finished product and an unfinished product and as such only provide an indication of what to expect.

Both Core (v0.13.2) and Bitcrust are currently running on the same machine (32gb/8 cores). CPU competition is prevented by not verifying at the same time, but competition for the OS cache is causing some extra variance.

Blocks only mode is used in Core and Bitcrust; the tx-index of Bitcrust is manually pruned to best match Core's defaults.
